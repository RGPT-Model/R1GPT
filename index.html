Architecture GPT Forward Pass

How GPT
Works

A systems-level walkthrough of the full transformer forward pass — from raw text to next-token prediction.

01 Input Text raw string
02 Tokenizer char → integer
03 Token IDs sequence of ints
04 Embeddings token + positional
05 Transformer Blocks × N stacked layers
06 Multi-Head Attention Q · Kᵀ / √d
07 Feed Forward Network 2× linear + ReLU
08 Layer Normalization pre-norm
09 LM Head linear projection
10 Next Token Prediction softmax → sample
Attention Mechanism
Attention(Q,K,V) = softmax(QKᵀ/√d_k)V. The scaled dot-product computes compatibility between every query and key pair. Masking ensures tokens only attend to previous positions — enabling autoregressive generation.
Residual Connections
Each transformer block uses residual skip connections: x = x + sublayer(x). This allows gradients to flow directly through the network depth, solving vanishing gradient problems in deep stacks.
Autoregressive Generation
At inference, the model generates one token at a time. Each generated token is appended to the context and fed back into the model. Temperature controls the sampling distribution — lower = deterministic, higher = creative.
Training Objective
Cross-entropy loss over next-token predictions. Given sequence x₁, x₂, ..., xₙ, the model maximizes P(xₜ | x₁...xₜ₋₁) across all positions simultaneously via teacher forcing.
